# GET /machines/{principalId}/alerts

**Source:** John Deere Operations Center - Machine Alerts API
**Endpoint ID:** `#/machines/{principalId}/alerts/get`

> Alerts

---

## Description

The alerts service allows clients to retrieve the information captured on the terminal when an alert occurred. For example, an alert is generated by a machine when the fuel pressure is low or there is a calibration fault in the fuel injector.Various alert types include machine, geofence, and maintenance. Response details will vary based on the alert type, which you can find in the links. Some responses will contain a "definition" that will give further information, see below. For each alert, the response links to the following resources:
machine: View machine information

Please Note - This API does not support eTags.

**OAuth Scope Required:** `eq1`

**Request URI**

```
GET https://sandboxapi.deere.com/platform/machines/{principalId}/alerts
```

**Accept:** `application/vnd.deere.axiom.v3+json`

## Parameters

| Parameter | Type | Description & Example | Default | In |
| --- | --- | --- | --- | --- |
| principalId<br>Required | string | Principal ID of the machine/equipment.<br>Example: 5432 | N/A | path |
| startDate | datetime | Returns alerts from a specified date onward. Requests are time-based with a maximum length of seven days.<br>Example: 2013-04-29T00:00:00Z | 24 hours before the timestamp of the request | query |
| endDate | datetime | Returns alerts until a specific date. Requests are time-based with a maximum length of seven days.<br>Example: 2013-05-02T00:00:00Z | The current time of the request | query |
| excludeAcknowledged | boolean | Excludes acknowledged alerts if "true."<br>Example: true | false | query |


## Response Details

| Field | Type | Description & Example |
| --- | --- | --- |
| id | string | Alert ID.<br>Example: 13243546 |
| time | datetime | Time of alert.<br>Example: 2014-10-21T 10:32:30.000Z |
| location | See sample code. | Location of machine during the alert.<br>Example: See sample code. Includes latitude and longitude. |
| color | string | Alert color1.<br>Example: YELLOW |
| severity | string | Severity of the alert1.<br>Example: MEDIUM |
| acknowledgementStatus | string | Shows whether the alert was acknowledged. Values are "Y" or "N".<br>Example: N |
| ignored | boolean | Shows whether the alert was ignored<br>Example: false |
| invisible | boolean | Shows whether the alert was invisible<br>Example: false |
| duration | object | Duration of the alert.<br>Example: See sample code. Includes "unit" and "valueAsInteger." |
| occurrences | integer | Number of times this alert has occurred.<br>Example: 1 |
| engineHours | object | Engine hours of the machine during the alert.<br>Example: See sample code. Includes "links," and "reading." |
| machineLinearTime | string | Machine's linear time during the alert.<br>Example: 97604307 |
| definition | See sample code. | Further details about the alert and alert source.<br>Example: See sample code. Includes "suspectParameterName", "failureModeIndicator", "bus", "sourceAddress", "threeLetterAcronym", "Id", and "description". |
